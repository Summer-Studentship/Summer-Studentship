# Dependency Licence Register v0.1

- **Document ID:** `SWE-ENV-LIC-WP1-REG-v0.1`
- **Work Package:** `SWE-ENV-LIC-WP1`
- **Baseline date:** 2026-07-21

**Decision state:** Initial dependency linking and redistribution decisions
accepted; every release must re-verify the exact shipped graph and artefacts.

## Register rules

This register applies to the dependency baseline in `vcpkg.json`, the external
components selected by the
[software stack decision register](../SWE-ENV-STACK/studentship_software_stack_decision_register_v0.1.md),
and transitives exposed by the current `x64-linux` dependency plan. It records
the compliance baseline; it is not a substitute for legal review when licence
terms, use mode or distribution change.

For every row:

- **source/acquisition** identifies the authoritative upstream terms and the
  approved route;
- **use/runtime** states whether code is header-embedded, linked, dynamically
  deployed or used only as an external tool;
- **release inclusion** means inclusion in the desktop application package,
  not merely use by a developer;
- linkage for vcpkg libraries is triplet/build-option dependent until a release
  link map and runtime tree are inspected; and
- `SWE-ENV-LIC` owns final classification in addition to the capability owner.

An external tool marked **exclude** must not be silently bundled. If a future
SDK, development container or installer redistributes it, that distribution
must carry the tool's own licence/source/notice obligations.

## Superseded and rejected acquisition routes

HighFive, RapidXML, direct Armadillo/FreeType/GLFW/OpenGL dependencies, Qt via
vcpkg, and the `sebsjames/mathplot` checkout are not selected components. They
must not appear in a release graph. Qt Quick 3D, Qt Quick Timeline and Qt Shader
Tools are deferred and likewise excluded unless an owning requirement and
module-specific licence review amend the baseline.

## Baseline redistribution decision

The selected baseline is licence-feasible under these controls:

1. permissive/MPL linked and header-embedded libraries carry their exact notices
   and required covered-source route;
2. open-source Qt uses only reviewed modules, dynamic linking and a complete
   LGPL/module-SBOM compliance bundle unless commercial terms are selected;
3. GPL/proprietary external applications are excluded and interact only through
   documented files/processes;
4. optional packages and their runtimes/notices are included only in releases
   that enable the corresponding feature; and
5. any unclassified component, dataset/grid, Qt module, runtime or transitive
   blocks release rather than inheriting an assumed licence.

The mandatory release procedure is defined in
`release_licence_verification_v0.1.md`.
